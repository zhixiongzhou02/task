## 1. 定位

本文件回答两个问题：

1. 如果要做“自然语言计划生成”，数据到底要准备哪些
2. 这些数据准备之后，应该怎么存、怎么用

## 2. 六类核心数据资产

建议优先准备：

1. 历史计划结构化样本
2. 商家筛选字段和枚举字典
3. 指标中心标准定义
4. 业务术语和别名词典
5. 计划 DSL / JSON Schema
6. 真实输入语句和人工标注结果

## 3. 历史计划结构化样本

目标：
作为历史经验库，用于关键词召回、向量召回、重排和 few-shot 参考。

数据来源：

1. 当前已有计划表
2. 历史作战计划记录
3. 历史计划配置快照
4. 历史计划效果数据

推荐字段：

1. 计划 ID
2. 计划名称
3. 计划描述
4. 创建时间
5. 计划周期
6. 业务目标类型
7. 商家筛选条件
8. 指标组合
9. 覆盖商家数
10. 达成率
11. 平均完成周期
12. 是否成功案例
13. 行业标签
14. 区域标签
15. 商家类型标签

推荐存储：

1. 关系表：`plan_history`、`plan_history_tags`、`plan_history_metric_group`、`plan_history_merchant_filter`、`plan_history_effect`
2. ES 索引：`idx_plan_history_search`
3. 向量层：历史计划描述、商家筛选摘要、指标组合摘要 embedding

## 4. 商家筛选字段和枚举字典

目标：
作为商家查询翻译字典，把自然语言转成外部商家系统支持的查询条件。

数据来源：

1. 商家系统接口文档
2. 字段定义
3. 标签定义
4. 查询口径约定

推荐字段：

1. 字段编码
2. 字段名称
3. 字段类型
4. 是否支持查询
5. 支持的操作符
6. 枚举值列表
7. 标签值列表
8. 字段说明
9. 同义词
10. 组合限制

推荐存储：

1. `merchant_filter_field`
2. `merchant_filter_operator`
3. `merchant_filter_enum`
4. `merchant_filter_alias`
5. `merchant_filter_constraint`

## 5. 指标中心标准定义

目标：
作为指标选择的合法边界，支撑指标意图映射、模板召回和冲突校验。

推荐字段：

1. 指标编码
2. 指标名称
3. 指标解释
4. 指标分类
5. 指标类型：实时 / 离线
6. 统计周期
7. 是否支持自动检核
8. 是否支持做主指标
9. 是否支持做过程指标
10. 数据来源
11. 口径版本
12. 同义词
13. 适用业务场景

推荐存储：

1. `metric_definition`
2. `metric_version`
3. `metric_category`
4. `metric_alias`
5. `metric_template`
6. `metric_conflict_rule`

## 6. 业务术语和别名词典

目标：
作为业务语言到系统语言的桥。

数据来源：

1. 历史计划名称和备注
2. 业务同学日常表达
3. 商家系统与指标系统术语

推荐字段：

1. 术语原文
2. 术语类型
3. 标准编码
4. 标准名称
5. 描述说明
6. 来源系统
7. 置信度
8. 是否启用

推荐存储：

1. `biz_term_dictionary`

## 7. 计划 DSL / JSON Schema

目标：
作为模型输出和系统落地之间的合同。

首期建议覆盖：

1. 计划元信息
2. 商家筛选条件
3. 指标组
4. 限制条件
5. 待确认项
6. 置信度

示意结构：

```json
{
  "plan_meta": {
    "plan_name": "",
    "goal_type": "",
    "time_range": ""
  },
  "merchant_filters": [],
  "metric_group": [],
  "constraints": [],
  "clarifications": [],
  "confidence": 0.0
}
```

推荐存储：

1. 代码仓库保存 Schema 文件
2. 数据库表保存版本信息：`plan_dsl_schema`、`plan_dsl_version`

## 8. 真实输入语句和人工标注结果

目标：
作为意图理解训练集和评估集。

推荐标注内容：

1. 目标类型
2. 商家筛选条件
3. 指标偏好
4. 时间范围
5. 限制条件
6. 缺失项
7. 是否需要澄清
8. 正确 DSL 草案

推荐存储：

1. `nl_plan_query_sample`
2. `nl_plan_query_label`
3. `nl_plan_query_eval_set`

## 9. 推荐存储分层

1. 在线关系库：
存历史计划结构化表、商家字段字典、指标定义、术语词典、标注样本、DSL 版本

2. Elasticsearch：
存历史计划检索索引

3. 向量检索层：
存历史计划和成功案例 embedding

4. 代码仓库：
存 DSL Schema、Prompt 模板和规则配置

## 10. 首期最小可行准备建议

1. 100 到 300 条高质量历史计划样本
2. 20 到 30 个常用商家筛选字段
3. 30 到 50 个高频指标
4. 100 到 200 个高频业务术语
5. 一版最小计划 DSL
6. 100 到 200 条人工标注输入样本

## 11. 如何进入系统链路

这六类数据会串成如下链路：

1. 术语词典先做归一化
2. 标注样本帮助意图理解
3. 历史计划做混合召回
4. 商家字段字典生成商家查询 DSL
5. 指标定义生成合法指标组
6. DSL 承接模型输出并做系统校验
