# 专项导航

返回：[导航与阅读说明](/Users/zhouzhixiong/code/zuozhanV2/docs/任务分配与自动检核系统AI方案/00-导航与阅读说明.md)

本专题只关注一件事：如何把管理岗的一句自然语言，最终转成结构化计划草案和可执行的商家筛选 DSL。

## 一、专题内层级关系

```text
自然语言计划生成
├── 00-专项导航
├── 01-数据准备与存储设计
├── 02-意图理解与筛选映射
├── 03-处理流程与时序设计
├── 04-数据资产落地实施设计
└── 05-接口清单与服务拆分
```

## 二、每份文档的职责

1. [数据准备与存储设计](/Users/zhouzhixiong/code/zuozhanV2/docs/任务分配与自动检核系统AI方案/02-自然语言计划生成/01-数据准备与存储设计.md)
定位：数据底座层。
回答：词典、历史计划样本、商家边界、指标术语从哪里来，怎么存。

2. [意图理解与筛选映射](/Users/zhouzhixiong/code/zuozhanV2/docs/任务分配与自动检核系统AI方案/02-自然语言计划生成/02-意图理解与筛选映射.md)
定位：语义理解层。
回答：自然语言如何一步步变成候选筛选意图，再映射成可执行 DSL。

3. [处理流程与时序设计](/Users/zhouzhixiong/code/zuozhanV2/docs/任务分配与自动检核系统AI方案/02-自然语言计划生成/03-处理流程与时序设计.md)
定位：系统编排层。
回答：模块如何串起来，在哪里校验，如何降级。

4. [数据资产落地实施设计](/Users/zhouzhixiong/code/zuozhanV2/docs/任务分配与自动检核系统AI方案/02-自然语言计划生成/04-数据资产落地实施设计.md)
定位：工程落地层。
回答：各类数据资产从原始数据到最终可用资产，中间经历哪些加工，以及存量和增量怎么跑。

5. [接口清单与服务拆分](/Users/zhouzhixiong/code/zuozhanV2/docs/任务分配与自动检核系统AI方案/02-自然语言计划生成/05-接口清单与服务拆分.md)
定位：接口实施层。
回答：自然语言计划生成链路应该拆成哪些服务、每个服务收什么、吐什么、失败时怎么处理。
